EP04	Gestión del Landing Pague <br>
EP05	Visualización de Zonas de Riesgo <br>

| Epic | titulo | descripcion |Criterios de Aceptacion|Relacionado con Epic ID|

|HU1|Registro de usuario| <br>As un cliente o comerciante <br>I want registrar mi mis datos <br>In order to poder ingresar a la aplciacion y recibir notificion. | **Esenario 1: Registro exitoso de usuario** <br> _Given_ que el cliente o comerciante accede a la pantalla de registro <br> _When_ ingresa correctamente sus datos (nombre, correo, contraseña, teléfono, etc.) y presiona el botón "Registrarse" <br> _Then_ el sistema valida la información y crea la cuenta exitosamente <br> _And_ el usuario es redirigido a la aplicación con acceso habilitado para recibir notificaciones <br><br> **Escenario 2: Error en el registro por datos inválidos** <br> _Given_ que el cliente o comerciante se encuentra en la pantalla de registro <br> _When_ ingresa datos incompletos, incorrectos o un correo ya registrado <br> _Then_ el sistema muestra un mensaje de error indicando el problema <br> _And_ no permite completar el registro hasta que los datos sean corregidos|EP01|
| HU2 | Inicio de sesión | <br>As cliente o comerciante, <br>I want iniciar sesión en la aplicación con mis credenciales <br>In order to acceder a mis funciones, recibir notificaciones y gestionar mi información de forma segura. | **Escenario 1: Inicio de sesión exitoso** <br> _Given_ que el usuario se encuentra en la pantalla de inicio de sesión <br> _When_ ingresa su correo y contraseña correctamente <br> _Then_ el sistema valida las credenciales <br> _And_ permite el acceso a la aplicación <br> _And_ redirige al usuario a la pantalla principal <br><br>  **Escenario 2: Error en el inicio de sesión por credenciales incorrectas** <br> _Given_ que el usuario está en la pantalla de inicio de sesión <br> _When_ ingresa un correo o contraseña incorrectos <br> _Then_ el sistema muestra un mensaje de error indicando credenciales inválidas <br> _And_ no permite el acceso a la aplicación |EP01|
| HU3 | Configuración de perfil | <br>As cliente o comerciante, <br>I want configurar y actualizar mi perfil <br>In order to mantener mis datos personales correctos y personalizar mi experiencia en la aplicación. | **Escenario 1: Actualización exitosa del perfil** <br> _Given_ que el usuario ha iniciado sesión en la aplicación <br> _When_ accede a la sección de perfil y modifica sus datos (nombre, teléfono, correo, etc.) correctamente <br> _Then_ el sistema guarda los cambios exitosamente <br> _And_ muestra un mensaje de confirmación <br><br> **Escenario 2: Error al actualizar el perfil por datos inválidos** <br> _Given_ que el usuario se encuentra en la sección de configuración de perfil <br> _When_ ingresa datos incorrectos o incompletos <br> _Then_ el sistema muestra un mensaje de error indicando los campos inválidos <br> _And_ no guarda los cambios hasta que la información sea corregida | EP01 |
| HU4 | Envío de alerta de emergencia | <br>As cliente o comerciante, <br>I want enviar una alerta de emergencia <br>In order to notificar rápidamente a otros usuarios y autoridades sobre una situación de riesgo. | **Escenario 1: Envío exitoso de alerta de emergencia** <br> _Given_ que el usuario ha iniciado sesión en la aplicación <br> _When_ presiona el botón de alerta y confirma el envío <br> _Then_ el sistema registra la alerta con la ubicación y hora <br> _And_ envía notificaciones a usuarios cercanos y contactos de emergencia | EP02 |
| HU5 | Geolocalización automática en alertas | <br>As cliente o comerciante, <br>I want que la aplicación obtenga automáticamente mi ubicación al enviar una alerta <br>In order to proporcionar información precisa del incidente. | **Escenario 1: Geolocalización obtenida correctamente** <br> _Given_ que el usuario ha iniciado sesión en la aplicación <br> _And_ tiene activado el servicio de ubicación en su dispositivo <br> _When_ envía una alerta de emergencia <br> _Then_ el sistema captura automáticamente la ubicación actual del usuario <br> _And_ adjunta la geolocalización a la alerta enviada  | EP02 |
| HU6 | Recepción de alertas cercanas | <br>As cliente o comerciante, <br>I want recibir alertas de emergencia cercanas <br>In order to estar informado y tomar precauciones ante posibles riesgos en mi entorno. | **Escenario 1: Recepción de alertas cercanas en tiempo real** <br> _Given_ que el usuario ha iniciado sesión en la aplicación <br> _And_ tiene activadas las notificaciones y la ubicación <br> _When_ se genera una alerta de emergencia dentro de su radio de proximidad <br> _Then_ el sistema envía una notificación inmediata al usuario <br> _And_ muestra los detalles de la alerta (ubicación, tipo de incidente y hora) | EP02 |
| HU7 | Cancelación de alerta | <br>As cliente o comerciante, <br>I want cancelar una alerta enviada <br>In order to evitar confusiones en caso de haberla activado por error. | **Escenario 1: Cancelación exitosa de alerta** <br> _Given_ que el usuario ha enviado una alerta de emergencia <br> _When_ accede a la opción de cancelar la alerta y confirma la acción <br> _Then_ el sistema actualiza el estado de la alerta a cancelada <br> _And_ notifica a los usuarios que recibieron la alerta sobre su cancelación | EP02 |
| HU8 | Historial de alertas | <br>As cliente o comerciante, <br>I want visualizar el historial de alertas <br>In order to consultar eventos pasados y llevar un seguimiento de las situaciones reportadas. | **Escenario 1: Visualización del historial de alertas** <br> _Given_ que el usuario ha iniciado sesión en la aplicación <br> _When_ accede a la sección de historial de alertas <br> _Then_ el sistema muestra la lista de alertas enviadas y recibidas <br> _And_ presenta detalles como fecha, ubicación, tipo de incidente y estado | EP02 |
| HU9 | Creación de reportes de incidentes | <br>As cliente o comerciante, <br>I want crear reportes de incidentes <br>In order to informar a la comunidad sobre situaciones relevantes que no requieren una alerta de emergencia. | **Escenario 1: Creación exitosa de reporte de incidente** <br> _Given_ que el usuario ha iniciado sesión en la aplicación <br> _When_ accede a la opción de crear reporte, completa la información del incidente y lo envía <br> _Then_ el sistema registra el reporte correctamente <br> _And_ lo publica para que otros usuarios puedan visualizarlo | EP03 |
| HU10 | Adjuntar evidencia a reportes | <br>As cliente o comerciante, <br>I want adjuntar evidencia (imágenes o videos) a mis reportes <br>In order to brindar mayor contexto y credibilidad al incidente reportado. | **Escenario 1: Adjuntar evidencia exitosamente** <br> _Given_ que el usuario ha iniciado sesión en la aplicación <br> _When_ crea o edita un reporte y adjunta una imagen o video válido <br> _Then_ el sistema carga correctamente el archivo <br> _And_ lo asocia al reporte para su visualización por otros usuarios | EP03 |
| HU11 | Visualización de reportes en el mapa | <br>As cliente o comerciante, <br>I want visualizar los reportes de incidentes en un mapa <br>In order to identificar rápidamente las zonas con mayor ocurrencia y tomar precauciones. | **Escenario 1: Visualización de reportes en el mapa** <br> _Given_ que el usuario ha iniciado sesión en la aplicación <br> _When_ accede a la vista de mapa de reportes <br> _Then_ el sistema muestra los reportes geolocalizados en el mapa <br> _And_ permite ver detalles básicos del incidente al seleccionar un punto | EP03 |
| HU12 | Filtro de reportes | <br>As cliente o comerciante, <br>I want filtrar los reportes de incidentes <br>In order to encontrar información relevante según mis necesidades (tipo de incidente, ubicación, fecha, etc.). | **Escenario 1: Filtrado de reportes exitoso** <br> _Given_ que el usuario ha iniciado sesión en la aplicación <br> _When_ accede a la sección de reportes y aplica filtros (tipo, fecha o ubicación) <br> _Then_ el sistema muestra únicamente los reportes que coinciden con los criterios seleccionados <br> _And_ actualiza la lista de resultados de forma dinámica | EP03 |
| HU13 | Descarga de la aplicación | <br>As visitante, <br>I want acceder a enlaces de descarga <br>In order to instalar la aplicación en mi dispositivo. | **Escenario 1: Descarga exitosa de la aplicación** <br> _Given_ que el usuario se encuentra en la landing page <br> _When_ selecciona el botón de descarga correspondiente a su dispositivo <br> _Then_ el sistema redirige a la tienda de aplicaciones correspondiente <br> _And_ permite iniciar la descarga de la aplicación | EP4 |
| HU14 | Visualización de información de la plataforma | <br>As visitante, <br>I want ver información clara sobre InstAlert <br>In order to entender sus beneficios y funcionamiento. | **Escenario 1: Visualización de información en la landing page** <br> _Given_ que el usuario accede a la landing page <br> _When_ navega por las secciones informativas <br> _Then_ el sistema muestra contenido sobre funcionalidades, beneficios y uso de la plataforma <br> _And_ presenta la información de manera clara y estructurada | EP4 |
| HU15 | Registro desde la landing page | <br>As visitante, <br>I want registrarme directamente desde la landing page <br>In order to comenzar a usar la aplicación rápidamente. | **Escenario 1: Registro exitoso desde la landing page** <br> _Given_ que el usuario se encuentra en la landing page <br> _When_ completa el formulario de registro con datos válidos <br> _Then_ el sistema crea la cuenta exitosamente <br> _And_ redirige al usuario a la aplicación o confirma el registro | EP4 |
| HU16 | Visualización de testimonios o casos de uso | <br>As visitante, <br>I want ver experiencias de otros usuarios <br>In order to confiar en la efectividad de la plataforma. | **Escenario 1: Visualización de testimonios** <br> _Given_ que el usuario accede a la landing page <br> _When_ navega a la sección de testimonios o casos de uso <br> _Then_ el sistema muestra opiniones y experiencias de usuarios <br> _And_ presenta contenido que refuerza la confianza en la plataforma | EP4 |
| HU17 | Visualizar mapa de zonas de riesgo | <br>As usuario, <br>I want ver un mapa con zonas de riesgo, <br>In order to identificar áreas peligrosas cercanas. | **Escenario 1: Visualización exitosa** <br> _Given_ que el usuario accede al mapa <br> _When_ la aplicación carga la información <br> _Then_ el sistema muestra zonas de riesgo geolocalizadas <br><br> **Escenario 2: Sin conexión** <br> _Given_ que el usuario no tiene conexión a internet <br> _When_ intenta acceder al mapa <br> _Then_ el sistema muestra un mensaje de error <br> | EP5 |
| HU18 | Filtrar zonas por nivel de riesgo | <br>As usuario, <br>I want filtrar zonas según nivel de peligro, <br>In order to analizar mejor la información. | **Escenario 1: Filtro aplicado** <br> _Given_ que el usuario selecciona un nivel de riesgo <br> _When_ aplica el filtro <br> _Then_ el sistema muestra solo las zonas correspondientes <br> **Escenario 2: Sin resultados** <br> _Given_ que no existen zonas con ese nivel <br> _When_ aplica el filtro <br> _Then_ el sistema muestra un mensaje informativo <br> | EP5 |
| HU19 | Consultar detalles de una zona de riesgo | <br>As usuario, <br>I want ver detalles de una zona específica, <br>In order to entender el tipo de incidentes ocurridos. | **Escenario 1: Consulta exitosa** <br> _Given_ que el usuario selecciona una zona en el mapa <br> _When_ accede a sus detalles <br> _Then_ el sistema muestra información como tipo de incidentes y frecuencia <br> **Escenario 2: Error de carga** <br> _Given_ que ocurre un fallo en la carga <br> _When_ intenta ver detalles <br> _Then_ el sistema muestra un mensaje de error <br> | EP5 |
| HU20 | Búsqueda de direcciones específicas | <br>As usuario, <br>I want buscar una dirección o lugar en el mapa <br>In order to conocer el nivel de riesgo de un destino antes de dirigirme allí. | **Escenario 1: Búsqueda con resultados** <br> _Given_ que el usuario ingresa una dirección en la barra de búsqueda <br> _When_ presiona "Enter" o el icono de lupa <br> _Then_ el sistema centra el mapa en esa ubicación y despliega los indicadores de riesgo cercanos <br> **Escenario 2: Dirección no encontrada** <br> _Given_ que el usuario ingresa un texto que no coincide con ninguna ubicación <br> _When_ realiza la búsqueda <br> _Then_ el sistema muestra un mensaje indicando que no se pudo encontrar el lugar <br> | EP5 |
| HU21 | Reporte de zonas con poca iluminación | <br>As usuario, <br>I want marcar áreas con deficiencia de alumbrado público en el mapa <br>In order to advertir a otros sobre condiciones que facilitan la delincuencia. | **Escenario 1: Registro de zona oscura** <br> _Given_ que el usuario selecciona la opción "Reportar falta de luz" <br> _When_ marca el punto en el mapa <br> _Then_ el sistema añade un icono específico de "Zona Oscura" visible para toda la comunidad <br> **Escenario 2: Visualización de advertencia** <br> _Given_ que un usuario navega cerca de un punto marcado como oscuro <br> _When_ el sistema carga el mapa <br> _Then_ muestra una advertencia visual sobre la falta de iluminación en ese tramo <br> | EP5 |
| HU22 | Recibir alertas de emergencia cercanas | <br>As usuario, <br>I want recibir alertas cercanas, <br>In order to estar informado de situaciones de peligro. | **Escenario 1: Notificación recibida** <br> _Given_ que ocurre una alerta cerca del usuario <br> _When_ el sistema detecta proximidad <br> _Then_ envía una notificación en tiempo real <br> **Escenario 2: Notificaciones desactivadas** <br> _Given_ que el usuario desactivó notificaciones <br> _When_ ocurre una alerta <br> _Then_ el sistema no envía notificación <br> | EP6 |
| HU23 | Configurar preferencias de notificación | <br>As usuario, <br>I want configurar qué tipo de alertas recibir, <br>In order to evitar información innecesaria. | **Escenario 1: Configuración guardada** <br> _Given_ que el usuario selecciona una zona en el mapa <br> _When_ accede a sus detalles <br> _Then_ el sistema muestra información como tipo de incidentes y frecuencia <br> **Escenario 2: Error de carga** <br> _Given_ que ocurre un fallo en la carga <br> _When_ intenta guardar <br> _Then_ el sistema muestra un mensaje de error <br> | EP6 |
| HU24 | Recibir notificaciones de actividad comunitaria | <br>As usuario, <br>I want recibir notificaciones sobre reportes de la comunidad, <br>In order to mantenerme informado. | **Escenario 1: Notificación de reporte** <br> _Given_ que un usuario reporta un incidente <br> _When_ este es relevante para la ubicación <br> _Then_ el sistema notifica <br> **Escenario 2: Sin relevancia geográfica** <br> _Given_ que el incidente está lejos <br> _When_ ocurre <br> _Then_ el sistema no notifica <br> | EP6 |
| HU25 | Enviar alerta a contactos de emergencia | <br>As usuario, <br>I want enviar alertas a mis contactos, <br>In order to pedir ayuda inmediata. | **Escenario 1: Registro exitoso** <br> _Given_ que el usuario activa el botón de emergencia <br> _When_ se genera la alerta <br> _Then_ el sistema envía notificación con ubicación a los contactos <br> **Escenario 2: Error de envío** <br> _Given_ que ocurre un fallo de red <br> _When_ intenta enviar la alerta <br> _Then_ el sistema muestra un mensaje de error <br> | EP7 |
| HU26 | Solicitar apoyo a la comunidad cercana | <br>As usuario, <br>I want solicitar ayuda a usuarios cercanos, <br>In order to recibir asistencia rápida. | **Escenario 1: Registro exitoso** <br> _Given_ que el usuario activa la solicitud de ayuda <br> _When_ se envía <br> _Then_ los usuarios cercanos reciben la alerta <br> **Escenario 2: Error de envío** <br> _Given_ que no hay usuarios en el área <br> _When_ envía la solicitud <br> _Then_ el sistema notifica que no hay disponibilidad <br> | EP7 |
| HU27 | Solicitar un asistente rapido | <br>As usuario, <br>I want solicitar ayuda a usuarios cercanos, <br>In order to recibir asistencia rápida. | **Escenario 1: Aceptación exitosa** <br> _Given_ que el usuario envía una solicitud de contacto <br> _When_ el destinatario hace clic en el enlace de aceptación <br> _Then_ el sistema activa el vínculo y notifica al usuario remitente <br> **Escenario 2: Rechazo de solicitud** <br> _Given_ que una persona recibe una invitación por error <br> _When_ selecciona "Rechazar" <br> _Then_ el sistema elimina la solicitud y no permite el envío de alertas a ese número <br> | EP7 |
| HU28 | Panel de monitoreo para contactos | <br>As contacto de confianza, <br>I want acceder a un panel de control cuando recibo una alerta <br>In order to visualizar la ubicación y estado del usuario en tiempo real. | **Escenario 1: Acceso al panel** <br> _Given_ que el contacto recibe un aviso de emergencia <br> _When_ abre el enlace en su navegador <br> _Then_ el sistema despliega un mapa dinámico con la ubicación del usuario y botones de llamada rápida <br> **Escenario 2: Sesión caducada** <br> _Given_ que el incidente fue resuelto hace más de 24 horas <br> _When_ el contacto intenta acceder al panel <br> _Then_ el sistema deniega el acceso por motivos de privacidad <br>| EP7 |

**Nota:** Historias de usuario (User Stories) de InstAlert organizadas por épicas y criterios de aceptación.
